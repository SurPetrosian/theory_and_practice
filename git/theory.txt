Git — система контроля версий, которая позволяет отслеживать историю изменений в файлах и одновременно работать над одним проектом многим разработчикам.

Репозиторий (от англ. repository — хранилище) — это виртуальное хранилище проекта. В нем можно хранить версии кода для доступа по мере необходимости.

Есть удаленный и локальный репозиторий.

Чтобы сохранить изменения на компе, разраб должен сделать commit

Коммит — это операция, которая берет все подготовленные изменения, они могут включать любое количество файлов, и отправляет их в репозиторий как единое целое.

git --version - проверка вверсии гита

добавляем свои данные из любой директории:
git config --global user.name "<имя фамилия>"
git config --global user.email "<ваш емейл>"

Working directory (рабочая директория) — это та папка, где лежат файлы, которые находятся под контролем Git.
Staging area, или индекс, — область, где лежат файлы, выбранные для коммита.

Главное в коммите — его атомарность, то есть он должен выполнять ровно одну задачу.

git init - создать репозиторий - директорию .git, содержащую все необходимые для работы git-файлы

git status - статус репозитория

Добавление в репозиторий происходит в два шага:
Первым шагом выполняется команда подготовки файлов:
git add <название_файла>

Теперь файл находится в состоянии «подготовлен к коммиту» или, другими словами, файлы попадают в индекс.
Делаем коммит:
git commit -m 'initial commit'
Флаг -m означает message, то есть описание коммита.

Рассмотрим, какие еще бывают состояния у файлов:
Staged — индекс, сюда переносятся файлы из untracked и modified. Если закоммитить файл, то он попадет в unmodified.
Modified — файл, который был изменен. Например, когда что-то дописали в коде.
Unmodified — мы уже сделали коммит, и файл сохранен в истории.

Файл unmodified не показывается в команде git status, пока не будет изменен. Git показывает только файлы untracked, modified и staged.

GitHub — это бесплатный (для одиночного использования) сервис, в котором хранят свои проекты большинство компаний и разработчиков.

Push — отправка новых коммитов на удаленный репозиторий, например в GitHub.

Pull — обновление локальной ветки за счет коммитов на удаленном репозитории.

cd .git - после инициализации гита создается папка, куда мы можем перейти

git add . - добавляем все файлы сразу в индекс рекурсивно

git rm --cached 'название файла' - удалить из индекса

git branch - посмотреть на какой ветке

git branch 'название ветки' - чтоб создать ветку

git branch -D 'название ветки' - чтоб удалить ветку

git checkout 'название ветки' - чтобы перейти на другую ветку

git checkout -b 'название ветки' - чтобы сразу создать и перейти на ветку
